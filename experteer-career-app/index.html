Motivation

Motivation

  • be present in the Apple and Google app markets
  • UX: better than web app in a browser
  • fast transition from the web architecture
  • share existing web-app frontend and backend
  • less native coding
  • benefit from native functions

Requirements

Requirements: 1st

  • The same (or better) as web user experience

  • Native functions:

    • Geolocation
    • Google analytics for Native apps
    • Marketing campaigns mgmt: Distimo tool
    • Next step: Push Notifications

Requirements: 2nd

  • Offline scenario
  • low-quality networks support
  • Smooth scrolling
  • Continuous scrolling
  • fast Job-search navigation
  • secure user's data/passwords
  • All app attributes: icons, launch-images

Limitations

Limitations

  • No upsell hooks ( iOS )
  • No payment ( iOS )
  • No premium functions/jobs for basic accounts

Idea

Let's

  • ... put existing web-app in a 'shell'

    that it looks and feels like a native app

  • ... implement a "Bridge" to access native functions

  • ... handle some init. tasks natively, e.g.

    • Language and Country version detection
    • Network request watch and react in case of problems

Use cases

Use case: Network unavailable

Show "offline page"

during that, query current network status and retry the remote request

Use case: Network slow

wait for remote request.

If data does not start loading within reasonable timeout,

show "offline page"

Use case: Network problem during ajax

show alert/notification in current context

Use case: remote page being loaded

show spinner during remote page transitions

Cordova Highlights

Apache Cordova is a set of device APIs that allow a mobile app developer to access native device functions

  • ... provides a platform-independend API to load a bundled or remote HTML content into WebView
  • ... provides a set of JavaScript libraries with device-specific native code
  • JavaScript APIs are consistent across multiple platforms
